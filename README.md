# 🎾 Tennis ATP Bot - Infrastructure & Deployment Guide

Данный документ содержит техническое описание инфраструктуры, стека и логики работы бота для поиска валуйных ситуаций в теннисных матчах ATP.

## 🏗 Инфраструктура
* **ОС**: Ubuntu (VPS в Нидерландах)
* **IP сервера**: `144.124.244.241`
* **Путь установки**: `/root/tennis-bot/`
* **Python**: 3.10+ (используется виртуальное окружение `venv`)
* **Менеджер процессов**: `PM2` (имя процесса: `tennis-bot`)

## 🗄 База данных (PocketBase)
Бот интегрирован в существующую экосистему PocketBase.
* **Версия**: 0.22.0
* **Локальный URL**: `http://127.0.0.1:8095` (для скриптов на сервере)
* **Внешняя админ-панель**: `https://v712746.hosted-by-vdsina.com/pb-polymarket/_/`
* **Авторизация**: Admin Auth (через переменные окружения)
* **Коллекция**: `signals`
    * `match` (Text): Названия игроков (A vs B)
    * `predicted_prob` (Number): Вероятность исхода (0.0 - 1.0)
    * `fair_odds` (Number): Справедливый коэффициент (1 / P)
    * `bookmaker_odds` (Number): Лучший найденный кэф в линии
    * `bookmaker_name` (Text): Название БК
    * `timestamp` (DateTime): Время генерации сигнала

## 💻 Технический стек
* **aiogram 3.x**: Асинхронный фреймворк для Telegram-бота
* **aiohttp**: Асинхронные HTTP-запросы к API и парсинг сайтов
* **BeautifulSoup4 + lxml**: Парсинг HTML-страниц (Scraping On-Demand)
* **numpy**: Математические расчеты (WMA, взвешенное среднее)
* **scipy.stats.binom**: Биномиальное распределение вероятностей
* **tenacity**: Логика автоматических ретраев при сбоях сети

## 🔌 API & Источники данных
1. **The-Odds-API**: Линия и коэффициенты (рынки `totals, h2h`).
2. **Open-Meteo API**: Прогноз погоды (ветер) по координатам турнира.
3. **TennisAbstract**: Динамический парсинг статистики игроков (Hold% / Return%) за последние 52 недели.

## 🧠 Алгоритм и Фильтры
1. **Покрытие**: Анализируются только быстрые покрытия (Hard, Grass, Indoor Hard, Carpet). Clay пропускается.
2. **Scraping On-Demand**: Бот парсит профиль каждого игрока из линии в реальном времени.
3. **WMA (Weighted Moving Average)**: Матчи, сыгранные за последние 60 дней от даты последнего матча в базе, получают вес **x2**, старые — **x1**.
4. **Expected Hold**: Расчет вероятности удержания подачи с учетом силы приема соперника. Порог входа — **>= 85%** для обоих игроков.
5. **Weather Kill Switch**: Пропуск открытых кортов при ветре **> 20 км/ч**.
6. **Value Math**: Расчет вероятности счета 5:5 в первом сете через биномиальную модель. Сигнал выдается при перевесе над линией букмекера более чем на **5% (EV+)**.

## 🚀 Команды управления
* **Старт**: `pm2 start main.py --name "tennis-bot" --interpreter ./venv/bin/python`
* **Перезапуск**: `pm2 restart tennis-bot`
* **Логи**: `pm2 logs tennis-bot`
* **Мониторинг**: `pm2 monit`

## ➕ Как добавить нового бота в эту систему
1. Используйте те же учетные данные PocketBase из `.env`.
2. Подключайтесь к `http://127.0.0.1:8095` для минимальной задержки.
3. Новые сигналы можно писать в ту же коллекцию `signals` или создать новую (например, `signals_soccer`).
4. Для изоляции используйте отдельную папку в `/root/` и собственное `venv`.
